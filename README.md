## в–ҡ Overview

**CORTE & CAOS** es una landing page single-page para una barberГӯa premium de MedellГӯn. El
objetivo del proyecto es sentirse como un sitio de agencia hecho a la medida вҖ” cero plantillas
genГ©ricas, cero tropos de barberГӯa vintage вҖ” con foco obsesivo en performance, accesibilidad y
una identidad visual coherente de principio a fin.

El flujo de reserva es una experiencia guiada de 4 pasos (Servicio вҶ’ Barbero вҶ’ DГӯa & Hora вҶ’
Datos) construida como UI, lista para conectarse a un backend de WhatsApp / Google Calendar. Al
confirmar, genera un _deep link_ de WhatsApp con el resumen de la cita.

### Secciones

## в–ҡ Tech Stack & decisiones de arquitectura

| TecnologГӯa                       | Por quГ©                                                                                                                                                                                                           |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Next.js 16** (App Router, RSC) | FCP casi instantГЎneo. Las secciones estГЎticas se renderizan como React Server Components; solo la interactividad (drawer, navbar, animaciones) baja como Client Components. Turbopack por defecto en dev y build. |
| **React 19.2**                   | Гҡltima generaciГіn del runtime que acompaГұa a Next 16.                                                                                                                                                             |
| **TypeScript (strict)**          | Cero `any`. El dominio (servicios, barberos, slots) estГЎ tipado en `src/types` y consumido en toda la app.                                                                                                        |
| **Tailwind CSS v4**              | ConfiguraciГіn basada en CSS (`@theme` en `globals.css`), sin `tailwind.config.js`. Design tokens como custom properties вҶ’ un Гәnico punto de verdad para color, tipografГӯa y motion.                               |
| **Framer Motion**                | Micro-interacciones y reveals on-scroll de alto rendimiento (transform/opacity), con respeto a `prefers-reduced-motion`.                                                                                          |
| **React Hook Form + Zod**        | ValidaciГіn type-safe del formulario de reserva. El schema de Zod es la fuente de verdad y deriva los tipos de TypeScript.                                                                                         |
| **Lucide React**                 | Iconos vectoriales limpios, tree-shakeable.                                                                                                                                                                       |
| **next/image**                   | ConversiГіn automГЎtica a WebP/AVIF, `srcset` responsive, `priority` en el hero para evitar CLS.                                                                                                                    |
| **next/font**                    | Self-hosting de Anton, Space Grotesk y JetBrains Mono con `display: swap` вҖ” cero requests a Google y cero layout shift.                                                                                           |
| **Vitest**                       | Tests unitarios de la lГіgica pura (formato COP, sanitizaciГіn, schema de reserva) sin arrastrar un runtime de navegador.                                                                                           |

### Principios de ingenierГӯa aplicados

- **Zero inline styles / zero inline handlers** en el markup вҖ” todo vive en Tailwind o en el
  sistema de diseГұo CSS.
- **Server-first**: las Client Components estГЎn acotadas a lo estrictamente interactivo.
- **Accesibilidad**: HTML semГЎntico (`main`, `section`, `article`, `figure`), skip-link, foco
  visible, `aria-*` en el drawer (role `dialog`, `aria-modal`), bloqueo de scroll y restauraciГіn
  de foco.
- **Seguridad** (ver abajo): CSP + cabeceras endurecidas, sanitizaciГіn de inputs y
  `rel="noopener noreferrer"` en todo enlace externo.
- **Testing**: la lГіgica pura (formato de precios, sanitizaciГіn, schema de reserva) estГЎ cubierta
  con Vitest y se ejecuta en cada push.
- **SEO**: Metadata API completa, OpenGraph + Twitter cards, imagen OG dinГЎmica, JSON-LD
  (`HairSalon`), `sitemap.xml` y `robots.txt` generados.

## в–ҡ Seguridad

- **Content Security Policy** estricta y cabeceras de seguridad definidas en
  [`next.config.ts`](./next.config.ts): `X-Content-Type-Options`, `X-Frame-Options`,
  `Referrer-Policy`, `Permissions-Policy` y `Strict-Transport-Security`.
- **SanitizaciГіn** de todo texto libre del formulario antes de que salga del cliente
  (`sanitizeText` en [`src/lib/utils.ts`](./src/lib/utils.ts)).
- **ValidaciГіn** en cliente con Zod: nombre, celular colombiano (regex) y notas con `maxlength`.
- **Sin secretos en el repo** вҖ” `.gitignore` cubre `.env*` desde el primer commit.

## в–ҡ Deployment

### Vercel (recomendado)

1. Sube el repositorio a GitHub.
2. Importa el proyecto en [vercel.com/new](https://vercel.com/new) вҖ” Next.js se detecta solo.
3. Define `NEXT_PUBLIC_SITE_URL` en las variables de entorno del proyecto.
4. Deploy. Las cabeceras de seguridad de `next.config.ts` se aplican automГЎticamente.

### Netlify

Usa el plugin oficial `@netlify/plugin-nextjs`. Build command `npm run build`, publish `.next`.

> **Nota sobre imГЎgenes:** el hero y la galerГӯa usan Unsplash vГӯa `next/image`. Los hosts
> permitidos estГЎn declarados en `images.remotePatterns` (`next.config.ts`). Para producciГіn con
> alto trГЎfico, considera migrar los assets a tu propio CDN.
